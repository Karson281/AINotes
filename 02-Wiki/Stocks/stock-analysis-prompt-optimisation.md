# 股票分析框架 Prompt 優化指令

> 以下係完整嘅優化指令，可直接複製俾任何 Hermes Agent 執行。
> 目標：將量化規則寫入 `/root/.hermes/skills/finance/stock-analysis-framework/SKILL.md`（或對應 skill file）。

---

## 第一部分：技術面量化規則（必須明確判斷，不得模糊表達）

### 【趨勢判斷】

- **強勢上升**：MA20 > MA50 > MA200，三線多頭排列
- **黃金交叉**：MA50 由下穿越 MA200（近 5 個交易日內）
- **死亡交叉**：MA50 由上穿越 MA200（近 5 個交易日內）
- **短期趨勢**：收盤價 > MA20 → 短線偏多；< MA20 → 短線偏空

### 【動能指標】

- **RSI 超買**：RSI > 70，配合成交量萎縮 → 警惕回調
- **RSI 超賣**：RSI < 30，配合成交量放大 → 留意反彈
- **RSI 中性**：30-70 之間，結合其他指標判斷

### 【MACD】

- **金叉**：MACD line 由下穿越 Signal line → 短線買入信號
- **死叉**：MACD line 由上穿越 Signal line → 短線賣出信號
- **柱狀體縮小**：動能減弱，趨勢可能轉變

### 【成交量】

- **放量上升**：成交量 > 20日均量 1.5倍，且收盤上升 → 強勢確認
- **縮量回調**：成交量 < 20日均量 0.7倍，且收盤下跌 → 正常整理
- **放量下跌**：成交量 > 20日均量 1.5倍，且收盤下跌 → 警告信號

## 第二部分：評級標準（必須輸出其中一個，不得輸出其他）

評級只可以係以下五個之一：

| 評級 | 條件 |
|------|------|
| **1. 建倉買入** | 技術面強勢，多項指標共振向上，基本面支持 |
| **2. 加倉買入** | 已持倉，技術面確認上升，可增持 |
| **3. 減倉賣出** | 技術面轉弱，出現死叉或放量下跌，建議減持 |
| **4. 清倉賣出** | 趨勢明確轉壞，多項指標惡化，建議全部離場 |
| **5. 觀望** | 信號不明確，或處於整固期，等待更清晰信號 |

**嚴禁輸出「睇好」、「睇淡」、「中性」等模糊評級。**

## 第三部分：輸出格式（每次分析必須包含以下所有項目）

```
📊 [股票代號] [股票名稱] — [日期]

【評級】建倉買入 / 加倉買入 / 減倉賣出 / 清倉賣出 / 觀望

【技術面】
- 趨勢：[MA 排列狀況]
- RSI：[數值] — [超買/超賣/中性]
- MACD：[金叉/死叉/中性]
- 成交量：[放量/縮量/正常]

【關鍵價位】
- 支撐位：[價格]
- 阻力位：[價格]
- 止損位：[價格]（建議入場價下方 X%）

【操作建議】
[具體可執行的建議，例如：「現價 $XX 附近分批建倉，止損 $XX，目標 $XX」]

【風險提示】
[具體風險，例如：「大市整體偏弱，建議控制倉位在 X% 以內」]
```

### 執行原則

1. **評級只可以係五個之一**：建倉買入 / 加倉買入 / 減倉賣出 / 清倉賣出 / 觀望
2. **嚴禁模糊評級**：不得輸出「睇好」、「睇淡」、「中性」等
3. **每個 section 都要有實際數值**：RSI 要寫數字，唔好剩係寫「中性」
4. **操作建議要可執行**：要有人名幣價格、百分比、具體動作
5. **風險提示要誠實**：冇信心就寫「信號不明確，建議觀望」

## 第四部分：宏觀過濾器（分析前必須考慮）

以下情況出現時，**所有評級自動降一級**（建倉→觀望，加倉→觀望）：

| 情境 | 觸發條件 | 降級影響 |
|------|----------|----------|
| FOMC 議息 | 聯儲局議息前後 3 個交易日 | 建倉/加倉 → 觀望 |
| 大市過熱 | 恒指 RSI > 75 | 建倉/加倉 → 觀望 |
| 市場恐慌 | 恒指單日跌幅 > 2% | 建倉/加倉 → 觀望 |
| 監管政策風險 | 中國重大監管政策公布後 48 小時內 | 建倉/加倉 → 觀望 |

### 實作要求

每次分析前，先用 `quote ^HSI` 攞恒指數據，檢查 RSI(14) 同單日變化。
觸發時在【風險提示】註明：**「宏觀過濾器觸發：XXXX」**
