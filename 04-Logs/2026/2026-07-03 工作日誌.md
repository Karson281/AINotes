## 📝 重要備忘
/root/.hermes/plans/migrate-away-from-mqtt-proma.md

  Hermes CLI / Telegram Gateway
            │
            ▼
        Kanban Board (取代 MQTT)
            │
            ▼
        Obsidian Vault

     

     好處：
     - 少一個 moving part（MQTT broker、proma-bot.py）
     - Kanban 係 Hermes 原生 feature，有 /kanban slash command + hermes kanban CLI
     - dispatch 模式可以自動 spawn worker agent 做任務

     壞處：
     - Kanban 係 pull-based（agent 要 check board），MQTT 係 push-based（即時通知）
     - 如果要即時 trigger（例如 Telegram 收到指令即刻開工），kanban 要配合 cron polling 或者 dispatcher daemon
     - 你現有嘅 proma 專屬 workflow 要全部 rewrite
     停用 proma-bot.py

     失去嘅功能：
     - MQTT → Obsidian 文件分發
     - Telegram bot（如果 Hermes Gateway 接手就冇問題）
     - 任何 proma-bot.py 度身訂做嘅 logic

     得到嘅：
     - Telegram polling conflict 自動消失（得返一個 bot instance）
     - 減少一個 process + codebase 要 maintain
     - 統一晒由 Hermes 做 frontend
     Google 全生態 + 股票分析嘅影響

     而家：  Telegram ── proma-bot.py ── Hermes
        （分兩層，有 conflict 風險）

     以後：  Telegram ── Hermes Gateway ── Kanban ── Worker Agents
        （一層，原生支援）
