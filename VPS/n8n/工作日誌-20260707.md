# n8n Stock Alert Workflow — 工作日誌

**日期：** 2026-07-07
**用時：** 約 4 小時（連 debug）

---

## 目標

建立一個每日 stock alert workflow（>2% 變動 → Telegram），監控 19 隻股票（14 HK + 5 US）。

## 最終架構

```
Cron (every hour)
  → Stock List (Code node, 19 stocks)
  → Fetch & Calc >2% (Code node, loop all + 2s delay)
  → IF >2%? (filter |Math.abs(cpct)| > 2)
  → Format Msg (Code node, nice Telegram format)
  → Send TG (Telegram node)
```

## 關鍵問題與解決方案

### 1. Database 寫入錯 target
- 最初所有修改寫入 `/root/vault/VPS/n8n/n8n.db`，但 Docker volume 嘅 database 喺 `/var/lib/docker/volumes/n8n_n8n_data/_data/database.sqlite`
- **教訓：** 一定要用 `docker inspect n8n | grep Mounts` 確認 volume path

### 2. SQLITE_CORRUPT
- Copy DB 時 WAL/shm 檔案衝突導致 corruption
- **Fix：** `PRAGMA journal_mode=DELETE` 停用 WAL mode

### 3. Code node HTTP request
- `$http` → not defined
- `fetch()` → not defined
- `require('https')` → disallowed（default sandbox）
- **Fix：** 加 `NODE_FUNCTION_ALLOW_BUILTIN=*` 到 docker-compose env

### 4. `putOutputInField` 唔 work
- HTTP Request node 嘅 `putOutputInField` 只喺 `responseFormat=text` 或 `file` 先有效，JSON format 唔支援
- **解決：** 放棄 HTTP Request node，用 Code node + `require('https')` 直接 fetch

### 5. Yahoo Finance rate limiting
- 19 個 rapid requests → "Too Many Requests"
- **Fix：** 每個 request 之間加 2s delay

### 6. IF node connection 掉轉
- `main[0]` = TRUE branch，`main[1]` = FALSE branch
- 之前掉轉咗 → FALSE items 去緊 Telegram，TRUE items 去咗空
- **Fix：** 互換 connection

### 7. Format Msg 走位
- `$input` 唔見咗 → `const item = .first().json;`
- 懷疑係 Python heredoc 或者 write_file 嘅 escaping issue
- **Fix：** 直接 Python update database
