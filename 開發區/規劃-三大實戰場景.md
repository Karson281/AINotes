# 規劃-三大實戰場景

## 場景 1 — 情報監控與本地觸發
**Hermes → Obsidian → Proma**

Hermes 在 VPS 執行 24/7 監控（RSS、價格、Server Logs），觸發後將情報寫入 `01-Inbox/` 並在 `03-Tasks/Pending/` 建立任務。Proma 監聽目錄，自動執行本地腳本（Telegram 通知、Python 程式等）。

**所需 infrastructure：**
- [ ] Hermes cron job 定時爬取/監控
- [ ] Hermes curl → Obsidian proxy → 寫入 01-Inbox/
- [ ] Proma 讀取 03-Tasks/Pending/ 自動執行
- [ ] Notification 機制

## 場景 2 — 深度研究與知識沉澱
**User → Obsidian → Hermes → Obsidian**

使用者在 `Pending/` 寫下研究題目，Hermes 讀取後進行深度網絡搜索，完成後將結構化報告寫入 `02-Wiki/`。Hermes 的學習循環會將此次研究轉化為 Skills。

**所需 infrastructure：**
- [ ] Hermes 讀取 Obsidian 03-Tasks/Pending/（已通）
- [ ] Hermes 深度搜索（deep research MCP / web search）
- [ ] Hermes 寫入 02-Wiki/（curl → proxy 已通）
- [ ] 學習循環轉化為 Skills（Hermes 原生功能）

## 場景 3 — 本地工作流調用全局記憶
**Proma → Obsidian**

Proma 在本地處理任務時，透過 MCP 直接搜尋 `02-Wiki/` 中由 Hermes 整理的長期記憶，實現「全局共享記憶」。

**所需 infrastructure：**
- [ ] Proma MCP 搜尋 Obsidian（proma-mcp + Obsidian REST API 已通）
- [ ] 02-Wiki/ 目錄結構化積累
- [ ] 跨 session 記憶查詢
